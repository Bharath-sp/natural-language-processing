== Introduction ==
Here we do group capture, find and replace.

. Understand the requirement: What needs to be replaced? What should be the replacement?
. Represent the patterns using regex. Enclose the pattern(s) that needs to be replaced with parentheses to segregate them into capture groups.
. Come up with a substitution pattern by using the captured pattern groups.

== Find and Replace ==
Say our requirement is to convert the input strings to:

image::.\images\regex_find_replace_01.png[align='left', 400, 300]

We use the regex `([0-9]+)x([0-9]+)` matches all the strings above. And here we have two capture groups. These capture groups help us to capture the pattern and assigns a number to it. The exact reference of capture groups can be done by \1, \2, ... etc. Then we come with a substitution pattern: `\1 pix by \2 pix`.

For these use cases, `sed` can used in Linux. `sed` command searches for a pattern, and replaces it with a substitution pattern.

----
sed -r 's/search_pattern/substitution_pattern/g' input_file_name
----

`-r` enables the POSIX extended set. `g` stands for global. By default, the `sed` command substitutes only the first match. If we add the `g` option, it will substitute ALL occurrences.

*Using Python:*

`re.sub()` is used for this purpose. This function takes: a search pattern, a substitution pattern, and a string. If the pattern isn't found, string is returned unchanged. Backreferences, such as \6, are replaced with the substring matched by group 6 in the pattern.

[source,python]
----
content='1280x720\n1920x1080\n1600x900\n1280x1024\n800x600\n1024x768\n'
1280x720
1920x1080
1600x900
...

re.sub(r'([0-9]+)x([0-9]+)', r'\1 pix by \2 pix', content)
'1280 pix by 720 pix\n1920 pix by 1080 pix\n1600 pix by 900 pix\n1280 pix by 1024 pix\n800 pix by 600 pix\n1024 pix by 768 pix\n'

#on printing
1280 pix by 720 pix
1920 pix by 1080 pix
1600 pix by 900 pix
...

content='7:32\n6:12\n12:23\n1:23\n11:33\n4:21\n'
7:32
6:12
12:23
...

print(re.sub(r'([0-9]{1,2}):([0-9]{2})', r'\2 mins past \1', content))
32 mins past 7
12 mins past 6
23 mins past 12
...
----

Escaping character is required only for search pattern. In the substitution string, we don't need to escape any symbols. The only thing we escape is the capture group number. \1 will get replaced with the contents of the first capture group.

[source,python]
----
content='914.582.3013\n873.334.2589\n521.589.3147\n625.235.3698\n895.568.2145\n745.256.3369\n'
914.582.3013
873.334.2589
521.589.3147
...

print(re.sub(r'[0-9]{3}\.[0-9]{3}\.([0-9]{4})', r'xxx.xxx.\1', content))
xxx.xxx.3013
xxx.xxx.2589
xxx.xxx.3147
...
----

To make a group non-capturing, we can use `(?:...)` instead of `(...)`.

[source,python]
----
content='Jan 5th 1987\nDec 26th 2010 \nMar 2nd 1923\nOct 1st 2008\nAug 3rd 2009\nJun 10th 2001\n'
Jan 5th 1987
Dec 26th 2010 
Mar 2nd 1923

pattern = r'([a-zA-Z]{3})\s([0-9]{1,2})(?:st|nd|rd|th)\s[0-9]{2}([0-9]{2})'
results = re.search(pattern, content)

results.group(1,2,3)
# ('Jan', '5', '87')

print(re.sub(pattern, r'\2-\1-\3', content))
5-Jan-87
26-Dec-10 
2-Mar-23

content='(914).582.3013\n(873).334.2589\n(521).589.3147\n(625).235.3698\n(895).568.2145\n'
(914).582.3013
(873).334.2589
(521).589.3147

pattern = r'\(([0-9]{3})\)(\.[0-9]{3}\.[0-9]{4})'
results = re.search(pattern, content)

results.group(1,2)
# ('914', '.582.3013')

print(re.sub(pattern, r'\1\2', content))
914.582.3013
873.334.2589
521.589.3147
----


== Group capture ==
The pattern r'\d{3}-\d{3}-\d{4}' is one solid group. We make this into multiple groups by enclosing the patterns in `()` and grab each group separately. We can also use `re.search()` and do:

[source,python]
----
text = "My telephone number is 408-555-1234"
phone_pattern = r'(\d{3})-(\d{3})-(\d{4})'

results = re.search(phone_pattern,text)

results.group() or results.group(0)
# '408-555-1234'

results.group(1) #'408'
results.group(2) #'555'
results.group(3) #'1234'
results.group(4) #index error
----
